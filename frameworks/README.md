# Frameworks Module

## 模块定位
`frameworks/` 用于承接已经完成输入整理与检查校验的信息，
把原始需求进一步转换为一版可执行的方案结构框架。

它不是最终输出文档，
而是方案生成前的中间结构层。

## 这一层要先回答的问题
在进入正式方案输出前，先回答：

- 这个方案到底围绕什么目标展开
- 主要路径是什么
- 关键步骤如何展开
- 权责接口如何划分
- 风险节点在哪里
- 节奏和时间如何安排
- 最后预期产出是什么

## 核心任务
本模块主要完成两件事：

1. 按母维度完成问题拆解
2. 形成一版结构框架

## 上游条件读取区
在进入本模块前，先读取 `inputs.md` 与 `checks.md` 中已经确认的条件，
避免框架层脱离上游信息单独生成。

### 1. 已确认条件
这些条件可以直接作为方案结构的成立基础。

示例格式：

- **condition_id**：COND-001
- **condition_type**：constraint / decision / exception
- **content**：
- **source**：
- **owner**：
- **status**：confirmed
- **impact_module**：framework / output

### 2. 待确认条件
这些条件暂时不能当成既定事实，
但必须在框架中标出影响位置。

示例格式：

- **condition_id**：COND-002
- **condition_type**：assumption / conflict
- **content**：
- **status**：pending
- **impact_module**：framework / output
- **risk_note**：

### 3. 本层默认假设
如果上游没有明确给出，但当前必须先假设才能继续推进，
必须在这里写明，而不能默认省略。

示例格式：

- **assumption_id**：
- **content**：
- **why_needed**：
- **risk_if_wrong**：
- **should_reconfirm**：yes / no

## 适用场景
本模块适用于：

- 方案设计前的结构搭建
- 商业项目的路径设计
- 营销与运营方案的框架生成
- 组织协作问题的结构重组
- 咨询类项目的方案骨架搭建
- AI辅助方案生成前的结构约束

### 条件驱动拆解原则
八个母维度不是脱离条件独立展开，
而是围绕“哪些条件已成立，哪些条件未成立”来展开。

处理要求：

- 每个母维度都要标出它依赖哪些上游条件
- 每个母维度都要标出是否存在未确认条件
- 如果某个维度的判断建立在默认假设上，必须显式记录
- 如果某个维度受例外条件影响，必须单独标注
- 如果某个维度因条件未确认而无法稳定成立，必须转人工关口

### 2.5 本层新增条件提炼
在母维度拆解和主线提炼后，
把框架层新产生的条件正式记录下来。

新增条件常见来源：

- 路径成立依赖的新前提
- 步骤先后顺序带来的前置条件
- 权责划分后暴露出的接口条件
- 时间安排带来的节奏条件
- 风险暴露后必须增加的止损条件

统一格式：

- **condition_id**：
- **stage**：framework
- **condition_type**：constraint / assumption / decision / exception
- **content**：
- **trigger_reason**：
- **impact_module**：output / review / case
- **status**：pending / confirmed
- **next_action**：继续生成 / 转人工关口 / 回退检查层
3. 形成一版结构框架

## 本模块输出
每次框架处理后，统一输出两部分：

- 一、母维度拆解
- 二、结构框架

## 文件说明
- `mother-dimensions.md`：八个母维度定义
- `structure-template.md`：标准框架模板

## 框架层放行结论
在进入 `outputs.md` 之前，
必须先判断本轮结构框架是否具备继续生成正式方案的条件。

### 1. 可直接进入 outputs
满足条件：
- 主线明确
- 七段式骨架完整
- 高优先级条件已确认
- 没有关键冲突悬而未决

### 2. 带条件进入 outputs
满足条件：
- 主线基本明确
- 存在少量待确认条件
- 不影响先产出第一版草案
- 已明确哪些部分只能写成“条件成立时”的版本

### 3. 不可进入 outputs
出现以下任一情况：
- 主线不明确
- 关键步骤依赖未确认条件
- 权责接口存在明显错位
- 高优先级冲突未解
- 方案路径建立在未经记录的默认假设上

### 4. 框架层输出结论
标准格式：

- **framework_status**：pass / conditional_pass / block
- **reason**：
- **confirmed_conditions**：
- **pending_conditions**：
- **new_conditions**：
- **should_trigger_human_gate**：yes / no
- **next_module**：outputs / checks / human_gate
