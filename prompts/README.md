# Prompts Module

## 模块定位
`prompts/` 用于管理方案设计工程化流程中的 AI 协作提示词。

它的目标不是一次性写出几个“万能提示词”，
而是建立一套可迭代、可分类、可复用、可校准的提示词模块。

也就是说，
`prompts/` 不是单纯的 prompt 收集页，
而是 AI 协作层的接口规范。

## 本模块解决的问题
1. 不同阶段需要什么类型的 AI 提示词
2. 提示词如何和工作流各模块对应
3. 提示词如何持续迭代补充
4. 重点词如何标注，避免 AI 误读主次关系

## 核心判断
AI 提示词不是“灵感句子”，
而是工作流中的结构化接口。

一个稳定的提示词模块，至少要做到：
- 能对应具体场景
- 能服务具体模块
- 能强调关键变量
- 能标注重点词
- 能随着案例与评审持续迭代

## 提示词分类
- `system-prompts.md`：系统级提示词
- `input-prompts.md`：输入分析提示词
- `check-prompts.md`：校验检查提示词
- `framework-prompts.md`：框架生成提示词
- `output-prompts.md`：输出草案提示词
- `review-prompts.md`：评审归因提示词
- `case-prompts.md`：案例沉淀提示词
- `pr-prompts.md`：PR说明提示词

## 重点词标注
所有关键提示词都建议显性标注重点词，
避免 AI 把边缘信息当主目标。

重点词建议优先覆盖：
- [主目标]
- [关键约束]
- [高风险]
- [责任归位]
- [输出格式]

## 提示词编写原则
1. 先定义任务，再给输出要求
2. 先给边界，再给发挥空间
3. 一个 prompt 尽量只服务一个主要动作
4. 输出格式尽量固定
5. 所有提示词都接受持续迭代
