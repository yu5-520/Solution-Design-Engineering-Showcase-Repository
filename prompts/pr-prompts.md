# PR Prompts

## Prompt 1：标准 PR 说明

### 1. 任务目标
- 你当前要完成的任务是：
根据本次仓库修改内容，生成一版清晰的 PR 说明。

### 2. 适用场景
- 本提示词用于：
新增模块、调整目录结构、修正模板、补充案例、更新提示词、修复流程问题时。

### 3. 重点词
- [主目标]：写清修改内容与修改原因
- [关键约束]：不能只写“优化了”
- [高风险]：只写改了什么，不写为什么改
- [责任归位]：要说明影响哪一层
- [输出格式]：背景 / 修改内容 / 修改原因 / 影响范围 / 验证方式

### 4. 执行要求
- 必须写修改背景
- 必须写具体修改项
- 必须写修改原因
- 必须写影响范围
- 必须写验证方式

### 5. 输出要求
- 输出必须包含：
1. 背景
2. 修改内容
3. 修改原因
4. 影响范围
5. 验证方式

### 6. 示例提示词
请根据以下代码仓库改动，生成一版 PR 说明。
不要只写“优化结构”或“补充内容”，
要明确：
1. 为什么改
2. 改了哪些文件或模块
3. 影响了工作流哪一层
4. 如何验证这次修改有效
请按以下格式输出：
1. 背景
2. 修改内容
3. 修改原因
4. 影响范围
5. 验证方式

## Prompt 2：目录迁移类 PR 说明

### 1. 任务目标
- 你当前要完成的任务是：
为“单文件 MVP → 目录结构 v2”的迁移生成 PR 说明。

### 2. 适用场景
- 本提示词用于：
目录结构升级、模块拆分、旧文件归档、README 导航更新。

### 3. 重点词
- [主目标]：说明迁移不是改逻辑，而是改承载
- [关键约束]：主链路不能被写成“重构全部流程”
- [高风险]：让人误以为逻辑大改
- [责任归位]：要说明哪些模块被迁移
- [输出格式]：背景 / 迁移内容 / 不变部分 / 风险提示 / 验证方式

### 4. 执行要求
- 必须强调主链路未改变
- 必须说明哪些旧文件迁到了哪些目录
- 必须说明旧文件如何归档
- 必须说明如何验证迁移不影响使用

### 5. 输出要求
- 输出必须包含：
1. 背景
2. 迁移内容
3. 不变部分
4. 风险提示
5. 验证方式

### 6. 示例提示词
请为本次仓库目录迁移生成 PR 说明。
重点说明：
1. 为什么要从单文件结构迁移到目录结构
2. 哪些模块被拆分到新目录
3. 哪些主链路保持不变
4. 迁移可能带来的使用风险
5. 如何验证迁移后仍可按 Input → Check → Framework → Output → Review → Case 跑通
