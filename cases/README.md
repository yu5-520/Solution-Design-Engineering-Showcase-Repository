# Cases Module

## 模块定位
`cases/` 用于沉淀成功案例与失败案例，
并将案例中的有效经验与失效原因，
转化为可回流到系统中的结构资产。

它不只是记录“发生了什么”，
更重要的是回答：

- 这次成功或失败，关键原因是什么
- 哪一层起了作用，哪一层失效了
- 这次案例能为系统新增什么
- 这些新增内容应该回流到哪个模块

也就是说，案例沉淀不是为了“留档”，
而是为了推动模板、检查点、框架、责任归因与风险清单持续进化。

## 核心判断
成功案例和失败案例都不是结果展示页，
而是系统升级的样本来源。

案例沉淀后，原则上可以沉淀出以下内容：

1. 新模板
2. 新检查点
3. 新案例
4. 新时间数据
5. 新责任归因样本
6. 新风险清单

## 适用场景
本模块适用于：

- 项目完成后的案例归档
- 方案执行后的复盘沉淀
- 成功路径抽象
- 失败路径止损归因
- 新场景进入系统时的经验补充
- 模板和检查机制迭代前的样本整理

## 案例分类
默认分为两类：

### 1. 成功案例
指目标基本达成、路径较稳定、执行结果可复用的案例。

### 2. 失败案例
指目标未达成、执行偏差较大、路径失效或中途止损的案例。

必要时也可标记为：
- 部分成功
- 部分失败
- 结构升级案例
- 纯框架样本

## 标准处理流程
1. 先识别案例类型
2. 还原关键链条
3. 提取关键变量
4. 完成归因
5. 转化为沉淀项
6. 明确回流位置

## 文件说明
- `templates/`：标准案例模板
- `success/`：成功或部分成功案例
- `failed/`：失败或止损案例
