# Reviews Module

## 模块定位
`reviews/` 用于承接方案输出后的评审、归因、偏差记录与版本修正。

它不负责重新生成方案，
而是负责回答两个更关键的问题：

1. 这版方案的问题到底出在哪
2. 这个问题应该回退到哪一层修正

也就是说，评审模块的任务不是简单打分，
而是把错误定位到具体层级，
避免团队陷入反复重写、反复返工、反复争论却找不到源头的状态。

## 核心判断
在本工作流里，错误不会无限发散。

错误只会落在几类地方：
1. 输入模板问题
2. 校验节点问题
3. 拆解维度问题
4. 框架模板问题
5. 输出格式问题
6. 审核机制问题

## 评审目标
本模块主要完成四件事：

1. 定位问题
2. 完成归因
3. 记录偏差
4. 形成修正建议

## 标准评审流程
1. 先看问题表现
2. 再做归因定位
3. 记录偏差
4. 形成修正建议

## 本模块输出
每次评审后，统一输出五部分：

- A. 问题描述
- B. 问题归因
- C. 偏差记录
- D. 修正建议
- E. 结论判断
